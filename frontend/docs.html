[ SECTION 01 · OVERVIEW ]

1. Genel Bakış

Monad Quantum Oracle, Monad ağı üzerinde çalışan akıllı kontratlara gerçek fiziksel kuantum rastgeleliği (QRNG) sağlayan, pull-based mimariye sahip bir altyapı protokolüdür. Avustralya Ulusal Üniversitesi'nin (ANU) kuantum laboratuvarındaki vakum elektromanyetik alan dalgalanmaları ölçülerek elde edilen entropi, off-chain olarak ECDSA (secp256k1) algoritmasıyla imzalanır ve Monad ağına tek blokta doğrulanarak aktarılır.

Neden Kuantum Entropisi?

Sözde rastgele sayı üreteçleri (PRNG — block.timestamp, keccak256, prevRandao vb.) madenciler veya validatörler tarafından manipüle edilebilir. Bu durum, adil çekiliş, NFT mint sıralaması veya DeFi tasfiye mekanizmaları gibi kritik süreçlerde ciddi güvenlik açıkları doğurur. Kuantum QRNG ise evrenin temel fizik yasalarını kullanarak %100 tahmin edilemez ve manipüle edilemez bir entropi kaynağı sunar. Hiçbir bilgisayar, kuantum bilgisayar dahil, bu değerleri önceden hesaplayamaz.

[ SECTION 02 · INSTALLATION ]

2. Kurulum

Gereksinimler (Prerequisites)

SDK'nın düzgün çalışabilmesi için aşağıdaki araçların sisteminizde kurulu olması gerekmektedir:

  • Node.js >= 18.0
  • Ethers.js v6
  • Bir Monad RPC endpoint (Testnet veya Mainnet)

Paket Kurulumu

Tercih ettiğiniz paket yöneticisine göre aşağıdaki sekmelerden birini kullanarak SDK'yı projenize ekleyin:

npm install kuantum-qrng-sdk ethers axios
yarn add kuantum-qrng-sdk ethers axios

[ SECTION 03 · QUICKSTART ]

3. Hızlı Başlangıç

SDK'yı projenize dahil edip bir Monad RPC sağlayıcısı ve işlem imzalayıcı cüzdan (Wallet) ile başlatmak oldukça basittir. Aşağıdaki örnek, SDK'nın tam yaşam döngüsünü — başlatma, entropi çekme ve on-chain doğrulama — göstermektedir:

import { QuantumSDK } from "kuantum-qrng-sdk";
import { ethers } from "ethers";

// ────────────────────────────────────────────────
// 1. Monad RPC Sağlayıcısı ve İmzalayıcı Cüzdan
// ────────────────────────────────────────────────
const provider = new ethers.JsonRpcProvider("https://testnet-rpc.monad.xyz");
const wallet = new ethers.Wallet("YOUR_PRIVATE_KEY", provider);

// ────────────────────────────────────────────────
// 2. SDK'nın Deploy Edilmiş Oracle Adresiyle Başlatılması
// ────────────────────────────────────────────────
const ORACLE_CONTRACT = "0xe7f1725E7734CE288F8367e1Bb143E90bb3F0512";
const oracle = new QuantumSDK(
  "https://testnet-rpc.monad.xyz",
  ORACLE_CONTRACT,
  "YOUR_PRIVATE_KEY"
);

console.log("✓ Quantum Oracle SDK başarıyla yapılandırıldı.");

// ────────────────────────────────────────────────
// 3. Kuantum Entropisi Çekme ve On-Chain Doğrulama
// ────────────────────────────────────────────────
async function requestQuantumEntropy() {
  try {
    const txHash = await oracle.fetchAndSubmitRandom();
    console.log(`Doğrulama İşlemi Onaylandı! Hash: ${txHash}`);
  } catch (error) {
    console.error("Oracle Hatası:", error.message);
  }
}

requestQuantumEntropy();

[ SECTION 04 · SOLIDITY INTEGRATION ]

4. Akıllı Kontrat Entegrasyonu

Monad üzerindeki dApp kontratınızın kuantum rastgeleliğinden faydalanabilmesi için ana QuantumOracle kontratının arayüzünü tanımlayıp lastRandomNumber() metodu ile en son doğrulanan değeri okuması yeterlidir.

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

// ──────────────────────────────────────────────
// Oracle Kontrat Arayüzü
// ──────────────────────────────────────────────
interface IQuantumOracle {
    function lastRandomNumber() external view returns (uint8);
    function consumeRandom(uint8 _randomNumber, bytes memory _signature) external;
    function oracleNode() external view returns (address);
}

contract QuantumLootBox {
    IQuantumOracle public immutable quantumOracle;
    
    mapping(address => uint8) public playerRewards;
    
    event LootBoxOpened(address indexed player, uint8 rewardId, uint8 quantumEntropy);

    constructor(address _oracleAddress) {
        quantumOracle = IQuantumOracle(_oracleAddress);
    }

    /// @notice Oyuncunun sandık açma fonksiyonu.
    /// Frontend önce oracle.fetchAndSubmitRandom() çağrısıyla
    /// kuantum verisini on-chain doğrulatmalıdır.
    function openLootBox() external {
        uint8 quantumEntropy = quantumOracle.lastRandomNumber();
        
        // 0-255 aralığındaki entropi → 1-10 arası ödül ID
        uint8 rewardId = (quantumEntropy % 10) + 1;
        playerRewards[msg.sender] = rewardId;

        emit LootBoxOpened(msg.sender, rewardId, quantumEntropy);
    }
}
Önemli Uyarı

Kontratınızın bu SDK'dan gelen işlemi kabul edebilmesi için, deploy sonrası oracleNode adresinin akıllı kontrata tanıtılması zorunludur. Kontrat sahibi (deployer) cüzdanınızla updateOracleAddress(relayerPublicAddress) fonksiyonunu çağırarak Vercel relayer'ın imza adresini kaydedin. Aksi halde tüm consumeRandom işlemleri 'Gecersiz Oracle Imzasi' hatası ile revert olacaktır.

[ SECTION 05 · API REFERENCE ]

5. API Referansı

fetchAndSubmitRandom()

SDK'nın ana fonksiyonudur. Vercel üzerindeki kuantum relayer sunucusuna HTTPS isteği atarak fiziksel entropi ve ECDSA imzasını çeker, ardından bu veriyi MetaMask veya programatik cüzdan aracılığıyla Monad akıllı kontratına gönderir.

async fetchAndSubmitRandom(): Promise<string>

Parametreler

Bu metot herhangi bir parametre almaz. Relayer URL'i ve kontrat adresi SDK constructor'ında tanımlanmıştır.

Geri Dönüş Değeri (Return Type)

İşlem başarıyla tamamlandığında Monad üzerindeki doğrulama işleminin transaction hash'ini (string) döndürür. Relayer'dan gelen ham veri aşağıdaki JSON yapısını içerir:

{
  "randomNumber": 189,
  "signature": "0xf2b2dd70c6a2e48a930ff28eb06dc366...",
  "oracleAddress": "0x011394599d3dd2d52c72Ebc285CE5df60c9b130A",
  "source": "ANU QRNG"
}

getLastRandomNumber()

On-chain'deki akıllı kontratı sorgulayarak en son doğrulanmış kuantum rastgele sayısını (uint8, 0-255) döndürür.

async getLastRandomNumber(): Promise<number>
const lastNumber = await oracle.getLastRandomNumber();
console.log(`Son Doğrulanan Kuantum Sayısı: ${lastNumber}`);

[ SECTION 06 · ERROR HANDLING ]

6. Hata Yönetimi

Ağ gecikmeleri, API rate limitleri veya kullanıcının MetaMask üzerinden işlemi reddetmesi gibi durumları yakalamak için aşağıdaki try-catch hata yönetimi şablonunu uygulamanız önerilir:

async function safeQuantumRequest() {
  try {
    console.log("Kuantum entropisi talep ediliyor...");
    const txHash = await oracle.fetchAndSubmitRandom();
    console.log(`İşlem Onaylandı: ${txHash}`);
    
  } catch (error) {
    // ── Kullanıcı MetaMask'ta işlemi reddetti ──
    if (error.code === "ACTION_REJECTED" || error.message.includes("user rejected")) {
      console.warn("İşlem kullanıcı tarafından iptal edildi.");
      return;
    }
    
    // ── Relayer sunucusuna bağlantı kurulamadı ──
    if (error.message.includes("ECONNREFUSED") || error.message.includes("Network Error")) {
      console.error("Relayer sunucusuna ulaşılamıyor. Lütfen bağlantınızı kontrol edin.");
      return;
    }
    
    // ── ANU API rate limit aşıldı (otomatik fallback devrede) ──
    if (error.message.includes("429")) {
      console.warn("Kuantum API istek limiti aşıldı. Otomatik fallback (qrandom.io) devrede.");
      return;
    }
    
    // ── Oracle imza uyuşmazlığı ──
    if (error.message.includes("Gecersiz Oracle Imzasi")) {
      console.error("Oracle adresi kontratta kayıtlı değil. updateOracleAddress() çağırın.");
      return;
    }
    
    // ── Beklenmedik hata ──
    console.error("Bilinmeyen Oracle Hatası:", error.message);
  }
}

[ SECTION 07 · SECURITY & QUANTUM ARCHITECTURE ]

7. Güvenlik & Kuantum Mimari

Monad Quantum Oracle, ölçeklenebilir ve kuantum dirençli bir gelecek vizyonuyla inşa edilmiştir. Mevcut MVP sürümü, işlemlerin hızlı onaylanabilmesi ve düşük gas tüketimi için endüstri standardı ECDSA (secp256k1) imza doğrulama protokolüyle çalışmaktadır.

V2 Mainnet Vizyonu — Trustless Mimari

Kalıcı Monad Mainnet sürümümüzle birlikte sistem tam anlamıyla trustless (güven gerektirmeyen) bir yapıya evrilecektir. Relayer sunucumuz ANU kuantum laboratuvarından HTTPS üzerinden veriyi çekerken arka planda zkTLS (TLSNotary / zkPass) teknolojisi kullanarak bir Sıfır Bilgi Kanıtı (ZK Proof) üretecektir. Bu kanıt, 'Bu sayıyı uydurmadım, gerçekten resmi qrng.anu.edu.au adresinden HTTPS ile çektim ve yolda değiştirmedim' gerçeğini kriptografik olarak kilitler. Akıllı kontratımız, laboratuvarın SSL/TLS sertifikasıyla mühürlenmiş bu ZK kanıtını doğrulayarak sistemi %100 trustless hale getirecektir. Ek olarak, post-quantum cryptography standartlarına uyum sağlamak adına ECDSA yerine Lamport Tek Seferlik İmzaları (Lamport OTS) devreye alınacaktır.

Bu evrimsel mimari sayesinde Monad Quantum Oracle, günümüzün en güçlü kuantum bilgisayarlarının bile kıramayacağı, fizik yasalarıyla korunan tam anlamıyla merkeziyetsiz bir rastgelelik altyapısı sunacaktır.
