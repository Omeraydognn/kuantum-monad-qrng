# Monad Quantum Randomness Oracle (QRNG)

> True physical entropy meets cryptographic proof on the Monad Network.

Monad QRNG is a pull-based, verifiable quantum randomness oracle. It retrieves physical entropy generated by vacuum fluctuations of electromagnetic fields from physical quantum sources (ANU & qrandom.io) off-chain, signs it using ECDSA (secp256k1) signatures, and delivers it to the Monad network where it is verified cryptographically in a single block.

This repository implements the entire end-to-end infrastructure, including the smart contract, off-chain signer relay node, developer SDK, and a Web3 portal dashboard.

---

## Repository Structure

*   **`contract/`**: Solidity smart contracts (`QuantumOracle.sol`), deployment scripts, and mocha unit test suites.
*   **`relayer/`**: Node.js Relayer Node that queries physical quantum devices and signs data using ECDSA.
*   **`sdk/`**: Source code of the official TypeScript SDK published on NPM as [`kuantum-qrng-sdk`](https://www.npmjs.com/package/kuantum-qrng-sdk).
*   **`frontend/`**: The pointillist interactive dashboard with a 3D WebGL particle Bloch sphere background, hosted on [Vercel](https://kuantum-sigma.vercel.app).

---

## Architectural Lifecycle

1.  **Generation**: Quantum vacuum fluctuations are measured off-chain, producing authentic, unpredictable physical random bytes.
2.  **Relay & Sign**: The Relayer server fetches the random number, hashes it, signs the hash using its private key, and exposes the payload.
3.  **SDK Fetch**: The client dApp pulls the signed payload via the TypeScript SDK.
4.  **Verification**: The client submits the payload to the Monad smart contract, which verifies the Relayer's signature using on-chain `ecrecover`. If valid, the entropy is registered.

## Quickstart

### 1. Installation
Clone the repository and install dependencies in the respective directories:
```bash
# Install relayer dependencies
cd relayer && npm install

# Install contract dependencies
cd ../contract && npm install

# Install frontend dependencies
cd ../frontend && npm install
```

### 2. Running Locally
Run a local Hardhat node, deploy the contract, and start the relayer:
```bash
# Start local chain node (Tab 1)
cd contract && npx hardhat node

# Deploy contract locally (Tab 2)
cd contract && npx hardhat run scripts/deploy.js --network localhost

# Start relayer (Tab 3)
cd relayer && npm start
```

### 3. Production Deployment
*   **Frontend & API**: The frontend is fully optimized to be deployed directly to Vercel as a unified static site + serverless function (see `/frontend/vercel.json` and `/frontend/api/`).
*   **Contract**: Deploy `QuantumOracle.sol` to the Monad Testnet / Mainnet.
*   **SDK**: Install the SDK globally in your project using:
    ```bash
    npm install kuantum-qrng-sdk ethers
    ```
